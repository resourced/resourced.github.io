ResourceD Master

Receive data from agents & query them easily.

Think of it as facts database, use it query server data across all hosts.

Get started in 4 easy steps:

1. Create S3 bucket and proper IAM credentials.

2. Create an application:

RESOURCED_MASTER_S3_ACCESS_KEY={access-key} \
RESOURCED_MASTER_S3_SECRET_KEY={secret=key} \
RESOURCED_MASTER_S3_BUCKET={bucket} \
/path/to/resourced-master app create {name}

3. Create access token and give it to the agent:

RESOURCED_MASTER_S3_ACCESS_KEY={access-key} \
RESOURCED_MASTER_S3_SECRET_KEY={secret=key} \
RESOURCED_MASTER_S3_BUCKET={bucket} \
/path/to/resourced-master token create {basic|admin} {app-id}

4. Start ResourceD Master:

RESOURCED_MASTER_S3_ACCESS_KEY={access-key} \
RESOURCED_MASTER_S3_SECRET_KEY={secret=key} \
RESOURCED_MASTER_S3_BUCKET={bucket} \
/path/to/resourced-master serv

Available for Linux and OSX. For more information, visit the GitHub page.

ResourceD Master is a single binary program. Just cURL and run. Current version: 1.0.0
